Why svgify-react?

Built for developers who need flexibility without bloat

Zero Runtime Dependencies

Generated components have no external dependencies. All utilities are copied to your output directory.

Framework Agnostic Styling

Works with Tailwind, CSS Modules, styled-components, or plain CSS. No styling opinions forced.

TypeScript & JavaScript

Full TypeScript support with proper type definitions. Optional JavaScript output for flexibility.

Flexible Usage Modes

Choose direct imports, string-based registry, or both. Optimize for your needs.

SVGO Optimization

Automatic SVG optimization with customizable SVGO configuration. Smaller file sizes, better performance.

Post-Generation Hooks

Run custom commands after generation. Format with Biome, Prettier, or any tool you prefer.

Configuration Reference

Complete configuration options for svgify.config.json

Required Fields

inputDir REQUIRED

Path to directory containing raw SVG files

"inputDir": "./raw-icons"
outputDir REQUIRED

Path where generated components will be saved

"outputDir": "./src/icons"

Optional Fields

iconMode default: "both"

Generation mode: "direct", "registry", or "both"

"iconMode": "both"
typescript default: true

Generate TypeScript (.tsx) or JavaScript (.jsx) files

"typescript": true
className default: "icon"

Base className added to all icon components

"className": "icon"
postGenerate default: ""

Command to run after generation (formatting, linting, etc.)

"postGenerate": "biome format --write src/"
svgoConfig optional

Custom SVGO configuration for SVG optimization

"svgoConfig": { "multipass": true }

Direct Import Mode

          
Import individual icon components directly. Best for tree-shaking and
            performance.


          

            
import { DashboardIcon, SettingsIcon } from './icons';

function App() {
  return (
    <div>
      <DashboardIcon size={24} className="text-blue-500" />
      <SettingsIcon width={32} height={32} color="#ff0000" />
    </div>
  );
}

Registry Mode (String-based)

          
Use icon names as strings. Perfect for dynamic icon selection and CMS
            integration.


          

            
import { Icon } from './icons';

function App() {
  const iconName = 'dashboard'; // Dynamic!
  
  return (
    <div>
      <Icon icon="dashboard" size={24} className="text-blue-500" />
      <Icon icon={iconName} width={32} color="#ff0000" />
    </div>
  );
}

Both Modes (Default)

          
Get the best of both worlds. Use direct imports for performance-critical areas
            and registry for dynamic scenarios.


          

            
import { Icon, DashboardIcon } from './icons';

function App() {
  // Direct import - best performance
  const header = <DashboardIcon size={24} />;
  
  // String-based - dynamic selection
  const menuIcons = ['dashboard', 'settings', 'user'].map(name => (
    <Icon key={name} icon={name} size={20} />
  ));
  
  return <div>{header}{menuIcons}</div>;
}
